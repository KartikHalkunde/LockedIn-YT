# Source Code Submission Guide (Mozilla)

This guide explains how to submit LockedIn source code for AMO review.

## Current build model

- **Single source of truth**: `src/` (all code lives here)
- **Build scripts**: `scripts/build.sh` and `scripts/build.ps1`
- **Build process**:
  - Firefox: Direct zip of `src/`
  - Chrome: Temporary folder created, Firefox-specific metadata removed, then zipped and cleaned up
- **Output folder**: `dist/`
- **Artifacts**:
   - `lockedin-v<version>-firefox.zip`
   - `lockedin-v<version>-chromium.zip`
   - `lockedin-v<version>-source.zip`

`<version>` is read from `src/manifest.json`.

## Do we need separate source submission?

LockedIn does not use bundlers or transpilers. Even when source submission is optional, upload `lockedin-v<version>-source.zip` for transparency and faster review.

## Build steps reviewers can run

### Linux/macOS
```bash
chmod +x scripts/build.sh
./scripts/build.sh
```

### Windows (PowerShell)
```powershell
.\scripts\build.ps1
```

After running, reviewers should find all artifacts under `dist/`.

## What the source package includes

The source ZIP generated by the build script includes:

- `src/` (single source of truth)
- `scripts/` (build automation)
- `project-docs/` (build documentation)
- Root docs (`README.md`, `CHANGELOG.md`, `PRIVACY.md`, `LICENSE`, etc.)

**Note**: The `chromium-build/` folder is **not** included because it's auto-generated during the build process.

## Suggested notes to reviewer

Use this in AMO reviewer notes:

```text
Build from source:
1) Run ./scripts/build.sh (or scripts/build.ps1 on Windows)
2) Artifacts are generated in dist/

Expected files:
- lockedin-v<version>-firefox.zip
- lockedin-v<version>-chromium.zip
- lockedin-v<version>-source.zip

Build details:
- Firefox package: Direct zip of src/
- Chrome package: src/ copied to temp folder, browser_specific_settings removed, then zipped
- Both use identical JavaScript/HTML/CSS code
- Only difference: Chrome manifest has Firefox-specific metadata removed

This project does not use bundlers/transpilers/minifiers.
Source is plain JavaScript/HTML/CSS and directly reviewable.
```

## Final checks before upload

- Build completes without errors
- `dist/lockedin-v<version>-firefox.zip` is the uploaded extension package
- `dist/lockedin-v<version>-source.zip` is attached when requested/recommended
- Version in listing matches `src/manifest.json`
